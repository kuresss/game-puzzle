# ゲームデザイン改善仕様書

**目的:** 「また遊びたくなる」「友達に勧めたくなる」ゲームにするための改善仕様  
**策定:** Claude Sonnet 4.6 × 3ラウンドのゲームデザイン相談（2026-04-20）

---

## 1. 最優先バグ修正（確実な問題）

### マインスイーパー: 初回クリック安全性（first-click safety）

**問題:** 最初のクリックで地雷を踏む可能性がある。これは「スキルと無関係な理不尽な死」でユーザーが即離脱する原因になる。業界標準として**最初のクリック周囲8マスには地雷を配置しない**のが必須。

**確認箇所:** `GAME4-minesweeper/src/gameCore.js` の地雷配置ロジック

**修正方針:**
```js
// 地雷配置時に safeCells（最初のクリック位置 + 隣接8マス）を除外する
function placeMines(rows, cols, count, firstRow, firstCol) {
  const safeCells = new Set();
  for (let dr = -1; dr <= 1; dr++) {
    for (let dc = -1; dc <= 1; dc++) {
      const r = firstRow + dr, c = firstCol + dc;
      if (r >= 0 && r < rows && c >= 0 && c < cols) {
        safeCells.add(r * cols + c);
      }
    }
  }
  // safeCells を除いた位置にのみ地雷を配置
}
```

**実装コスト:** 小 | **効果:** 確実（初回クリック死亡によるストレス離脱を防ぐ）

## 2. リテンション強化機能（また来たくなる）

### 2-1. 「惜しかった」フィードバック

**背景:** 「もう一回！」の最大トリガーは「後悔の解消（あと少しで勝てた）」。現状は結果だけ表示されており、悔しさのフックが弱い。

**実装方針:** ゲームオーバー時のモーダルに「惜しかった情報」を1行追加する。

| ゲーム | 「惜しかった」の定義 |
|--------|-------------------|
| 2048 | 「最高タイル: 1024 / あと1回の合体で2048でした」 |
| マインスイーパー | 「残り3マスで地雷を踏みました」 |
| スネーク | 「長さ18 / 自己ベストまであと3」 |
| リバーシ | 「38-26で敗北 / 最後の角を逃しました」 |
| ハノイ | 「最適手数の1.5倍 / 次はあと○手削れます」（実装済みを活用） |
| コネクトフォー | 「あと1手で4つ揃いでした」 |

**実装コスト:** 小（各ゲーム5〜10行） | **効果:** 確実（リプレイ率向上）

---

### 2-2. 難易度の事後提案

**背景:** 初心者はどの難易度が自分に合うか分からない。事前選択より、プレイ結果を見た後の提案が効果的。

**実装方針:** クリア/ゲームオーバーモーダルに条件付きメッセージを追加する。

```
クリア + easy → 「easy をクリア！次は normal に挑戦しますか？」
連続2回敗北 + hard → 「normal に戻してみましょうか？」
初プレイ → 「easy から始めることをおすすめします」
```

**実装コスト:** 小（ゲームごと10行） | **効果:** 推測（難易度ミスマッチによる離脱を減らす）

### 2-3. デイリーチャレンジ（サブゴール型）

**背景:** 「毎日ゲームを開く理由」がない。Wordleが成功した最大の理由はこれ。PWA対応済みなので特に重要。

**実装方針（アカウント不要・サーバー不要）:**

```js
// 今日の日付 → ハッシュ → チャレンジ内容を決定論的に選択
function getTodayChallenge(gameName) {
  const dateStr = new Date().toISOString().slice(0, 10); // "2026-04-20"
  const seed = hashCode(dateStr + gameName);
  return CHALLENGES[gameName][seed % CHALLENGES[gameName].length];
}
```

**チャレンジ例:**

| ゲーム | チャレンジ内容 |
|--------|--------------|
| 2048 | 「512タイルを3分以内に作れ」 |
| マインスイーパー | 「normal を2分以内にクリアせよ」 |
| スネーク | 「長さ20以上を10秒間維持せよ」 |
| ハノイ | 「5枚を最適手数ちょうどでクリアせよ」 |
| リバーシ | 「対AI（normal）で40石以上取れ」 |

**クリア後の共有テキスト（Wordle形式）:**
```
🎮 今日の2048チャレンジ クリア！
512タイル: 2分18秒
https://kuresss.github.io/game-puzzle/
```

**実装コスト:** 中（各ゲーム30〜50行 + ポータル連携） | **効果:** 推測（毎日のリターン動機付け）

## 3. 回遊率向上（13本を全部遊んでもらう）

**背景:** 13本は現状「選択肢過多で弱み」。適切な導線設計で「毎日来る理由が多い強み」に変えられる。

### 3-1. 未プレイバッジ

**実装方針:** ポータルの各ゲームカードに「NEW」バッジを表示。1度でもプレイしたら消える。

```js
// ポータル index.html に追加
const GAME_KEYS = ['GAME3-2048', 'GAME4-minesweeper', ...];
GAME_KEYS.forEach(key => {
  const played = !!localStorage.getItem(`${key}_played`);
  if (!played) document.querySelector(`[data-game="${key}"]`)
    ?.classList.add('unplayed');
});

// 各ゲームのscript.jsに追加（ゲーム開始時に実行）
localStorage.setItem('GAME3-2048_played', '1');
```

```css
.game-card.unplayed::before {
  content: 'NEW';
  position: absolute;
  top: 8px; right: 8px;
  background: #f59e0b;
  color: #000;
  font-size: 0.7rem;
  font-weight: 800;
  padding: 2px 6px;
  border-radius: 4px;
}
```

**実装コスト:** 小（20行） | **効果:** 確実（「まだ遊んでいないゲームがある」という視覚的引力）

### 3-2. ゲーム推薦チェーン

**背景:** 全ゲームを並べるより「1本だけ次を提案」する方が行動につながりやすい（選択のパラドックス回避）。

**推薦マップ（各ゲームのクリア/ゲームオーバー画面に1本だけ表示）:**

| 今のゲーム | 次の推薦 | 理由 |
|----------|---------|------|
| 2048 | フラッドフィル | 数字→色に変わる「埋めるパズル」系 |
| スネーク | ブロック崩し | 反射神経・動体視力系 |
| ハノイの塔 | ライツアウト | 純粋な論理パズル系 |
| マインスイーパー | ヒットアンドブロー | 論理的推論で答えを絞る系 |
| リバーシ | コネクトフォー | CPU対戦・陣取り系 |
| コネクトフォー | マスターマインド | 配置を読む系 |
| 神経衰弱 | サイモン | 記憶力系 |
| サイモン | モグラたたき | 反射・瞬発力系 |

**実装コスト:** 小（推薦マップをJSオブジェクトで定義 + モーダルに1行追加） | **効果:** 推測（回遊率改善）

## 4. 各ゲーム個別の改善ポイント

### 2048
- **現状の良い点:** Undo 1回制限は正しい設計。2回以上にしると「負けの重み」が消える
- **改善:** タイムアタックモード（easy限定で「○分以内に2048」）を追加すると初心者の目標が明確になる

### マインスイーパー
- **最重要:** first-click safety（Section 1）
- **改善:** 盤面確定時刻（思考時間）をスコアに組み込む → 上級者の差別化指標になる

### スネーク
- **現状の問題:** スピードを上げるだけでは「速いだけで難しくない」体験になる
- **改善:** 高スコア時（長さ15以上）に「壁が縮む」など構造的な変化を追加すると緊張感が持続する

### リバーシ
- **問題:** AIが何を考えているか分からず初心者が一方的に負ける
- **改善1:** easy AIに意図的なランダム性を追加（推測: 30%の確率で最善手以外を選ぶ）
- **改善2:** 「角が重要」などのヒントをゲームオーバー時に1言表示するだけでリプレイ率向上

### ハノイの塔
- **現状の良い点:** 最適手数比較・移動評価は実装済み（強み）
- **問題:** 「法則の発見」前に飽きる初心者がいる
- **改善:** 最初の1回（3枚）クリア後に「n枚 = 2ⁿ-1 手の法則があります。次は4枚で確認してみよう」と法則をヒント表示する
