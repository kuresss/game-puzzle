## 1. 最優先バグ修正（確実な問題）

### 1-1. マインスイーパー：初回クリック安全性の確認と保証

**現状**  
`gameCore.js:40` の `placeMines(board, safeIndex, ...)` は `safeIndex` を地雷候補から除外する設計になっている。
しかし `script.js:191` で `placeMines` が呼ばれるタイミング・`safeIndex` に渡される値を目視確認し、**最初のクリックセルと隣接8マスが確実に安全かどうか**を検証する必要がある。

**問題の深刻度:** 高（初手爆死はゲーム放棄の最大要因）

**確認・修正手順**
1. `script.js` 内で `placeMines` を呼ぶ箇所を特定し、`safeIndex` が「プレイヤーが最初にクリックしたインデックス」であることを確認
2. 安全範囲を「クリックした1マス」→「クリックした1マス＋隣接8マス」に拡大する
3. `safeIndex` の代わりに `safeSet: Set<number>` を受け取れるよう `placeMines` シグネチャを拡張する

```js
// gameCore.js 修正イメージ
export function placeMines(board, safeSet, mines, rows, cols, random = Math.random) {
  const safeIndices = safeSet instanceof Set ? safeSet : new Set([safeSet]);
  const candidates = board.map((_, i) => i).filter(i => !safeIndices.has(i));
  // ... 以降変わらず
}

// script.js 呼び出し側
function buildSafeSet(index, rows, cols) {
  const set = new Set([index]);
  const r = Math.floor(index / cols), c = index % cols;
  for (let dr = -1; dr <= 1; dr++) for (let dc = -1; dc <= 1; dc++) {
    const nr = r + dr, nc = c + dc;
    if (nr >= 0 && nr < rows && nc >= 0 && nc < cols)
      set.add(nr * cols + nc);
  }
  return set;
}
```

## 2. リテンション強化機能（また来たくなる）

### 2-1. デイリーチャレンジ（サブゴール型）

**設計思想**  
「今日やること」という外部トリガーを作る。ログインやアカウントは不要で、**日付シードによる固定問題**で実現できる。

**対象ゲーム:** 2048・フラッドフィル・ハノイの塔・ヒットアンドブロー・マスターマインド（再現性のあるゲーム）

**仕様**

```
毎日0時リセット（ローカル日付ベース）
シード = YYYYMMDD の数値をゲーム別乱数シードに使用
クリア済みフラグ = localStorage に当日日付+ゲームIDで保存
表示 = ゲームトップに「📅 今日のチャレンジ」バナー
クリア時 = 「今日のチャレンジ達成！」モーダル（SNS共有ボタンは任意）
```

**表示する情報（モチベーション維持のため）**
- 達成: ✅ 今日のチャレンジ達成！
- 未達成: 📅 今日のチャレンジ（残り HH:MM）

| 実装コスト | 期待効果 |
|-----------|---------|
| 中（シード乱数の導入、UI追加） | 推測（習慣化の起点になれば高い） |

**実装メモ:** 現行ゲームの乱数を `Math.random()` から `seededRandom(seed)` に切り替えるためにゲームごとの対応が必要。既存テストが壊れないよう `random` 関数を引数で受け取る設計になっているゲームは容易（マインスイーパーはすでに対応済み）。

### 2-2. 「惜しかった」フィードバック

**設計思想**  
「あと少し」という感覚を数値で見せることで「もう一回」を誘発する。  
Round 2 の知見：**「もう一回」の最大トリガーは後悔の解消**。

**対象ゲームと「惜しかった」の定義**

| ゲーム | 惜しかった判定 | 表示メッセージ例 |
|--------|-------------|--------------|
| 2048 | 最大タイル ≥ 目標の50%（例: 1024到達） | 「惜しい！あと1合体で2048でした」 |
| マインスイーパー | 開けたマス数 ≥ 全安全マス×80% | 「あと{N}マスで全クリアでした！」 |
| フラッドフィル | 残り手数0時の塗り率 ≥ 85% | 「塗り残し{N}マス！手数+2ならクリアでした」 |
| ヒットアンドブロー | 最終試行でHit数3以上（4桁の場合） | 「最後の予想でHit 3！あと1歩でした」 |
| スネーク | スコアが自己ベストの70%以上 | 「自己ベストまであと{N}点！」 |
| ブロック崩し | 残りブロック数 ≤ 10 | 「あと{N}個で全クリアでした！」 |

**表示タイミング:** ゲームオーバー/失敗モーダルの既存テキストの下に1行追加するだけ。

| 実装コスト | 期待効果 |
|-----------|---------|
| 小（各ゲーム2〜5行の条件追加） | 確実（心理的な「もう一回」誘発） |

### 2-3. 難易度事後提案

**設計思想**  
「4難易度は方向性は正しいが事後提案がない」という知見を受けた改善。  
プレイヤーが「強い」「弱い」を自己申告する前に、**ゲーム結果から自動推薦する**。

**トリガーと提案内容**

```
クリア時（勝利）
  → 現在難易度が easy/normal/hard の場合: 「上の難易度に挑戦しますか？」ボタン
  → 現在難易度が oni の場合: 「他のゲームに挑戦！」（→ 後述の推薦チェーンへ）

ゲームオーバー時（連続2回以上）
  → 現在難易度が normal/hard/oni の場合: 「難易度を下げてみますか？」ボタン
  → ワンクリックで即切り替え＆リスタート

タイムアウト/手数切れ時（フラッドフィル等）
  → 同上
```

**実装詳細**

```js
// 各ゲームの結果モーダル内に追加
function showPostGameSuggestion(result, currentDifficulty, consecutiveLosses) {
  const levels = ['easy', 'normal', 'hard', 'oni'];
  const idx = levels.indexOf(currentDifficulty);
  
  if (result === 'win' && idx < 3) {
    showUpgradeBanner(levels[idx + 1]);
  } else if (result === 'lose' && consecutiveLosses >= 2 && idx > 0) {
    showDowngradeBanner(levels[idx - 1]);
  }
}
```

連続敗北カウントは `sessionStorage` で管理（永続化不要）。

| 実装コスト | 期待効果 |
|-----------|---------|
| 小〜中（各ゲームの結果モーダル改修） | 確実（離脱防止・ステップアップ促進） |

### 3-2. ゲーム推薦チェーン

**設計思想**  
「13本は現状弱み、推薦チェーンで強みに変えられる」という知見の具体化。  
ゲーム終了後に「次に遊びたくなる1本」を意味的につないで提示する。

**推薦マップ（意味的な類似性・難度差で設計）**

```
パズル（15ピース）   → ライツアウト    （空間パズル同士）
ライツアウト         → フラッドフィル  （色/状態変化パズル）
フラッドフィル       → 2048           （手数制限の緊張感）
2048                → ハノイの塔     （手順最適化パズル）
ハノイの塔           → リバーシ       （思考系・CPU対戦）
リバーシ             → コネクトフォー  （CPU対戦・陣取り）
コネクトフォー       → ヒットアンドブロー（論理推理）
ヒットアンドブロー   → マスターマインド （同ジャンル上位版）
マスターマインド     → 神経衰弱        （記憶系へ転換）
神経衰弱             → サイモン        （記憶系・反射）
サイモン             → モグラたたき    （反射系）
モグラたたき         → スネーク        （動き系）
スネーク             → ブロック崩し    （アクション系）
ブロック崩し         → マインスイーパー （集中系・達成感）
マインスイーパー     → パズル（15ピース）（一巡）
```

**UI:** ゲーム終了モーダルの下部に「次はこれ → 🃏 神経衰弱」ボタンを1つだけ表示。シンプルに1本のみ提示する（選択肢を増やさない）。

| 実装コスト | 期待効果 |
|-----------|---------|
| 小（データ定義 + ボタン追加） | 推測（推薦の精度次第で高くなる） |

## 4. 各ゲーム個別の改善ポイント

### 4-1. 2048 — ゲームオーバー時の「惜しかった」＋次手ヒント

**現状の問題:** ゲームオーバー時に最終状態がどれだけ惜しかったかわからない。

### 4-2. マインスイーパー — first-click safety 拡張 + タイマー表示の感情的利用

**現状の問題:**
1. first-click safety が「クリックした1マス」のみの可能性（要確認・修正は §1-1 参照）
2. タイマーが純粋な時計として機能しているが、「速さ」の達成感に活用されていない

**改善案**
- クリア時に「自己ベスト: {時間}秒」を明示（難易度別にlocalStorageで管理）
- 自己ベスト更新時は強調アニメーション（CSSのみ）
- フラグ数カウンター（残り地雷数）をより見やすい位置に移動検討

| 実装コスト | 期待効果 |
|-----------|---------|
| 小〜中 | 確実（タイムアタック欲求の醸成） |

### 4-3. スネーク — 速度可視化と「惜しかった」

**現状の問題:** ゲームオーバー時にスコアしか表示されず、何が良くて何が悪かったかわからない。

**改善案**
- ゲームオーバー時: 「長さ{N}・速度{レベル}で終了」を表示
- 自己ベストとの比較を1行で表示（「前回より+5点！」）
- 鬼モード（70ms・壁なし）は「壁衝突/自己衝突」の別カウントを表示し、死因を可視化

| 実装コスト | 期待効果 |
|-----------|---------|
| 小 | 推測（分析欲求による再挑戦） |

### 4-4. リバーシ / コネクトフォー — 敗因の可視化

**現状の問題:** CPU に負けたとき「なぜ負けたか」がわからず学習機会がない。

**改善案（リバーシ）**
- ゲーム終了後に「あなたの最悪手」を1手だけ表示（最も石差が広がった手）
- 表示: 「{N}手目でここに置いたのが痛かった」→ ボードをフラッシュ強調

**改善案（コネクトフォー）**
- CPU の「詰み手」が発生した瞬間に「詰み発生」テキストを一瞬表示（0.5秒後に消す）
- ゲームオーバー時に「CPU が{N}手目で詰みを作っていました」を表示

| 実装コスト | 期待効果 |
|-----------|---------|
| 中（ゲームロジックへの分析処理追加） | 推測（「もう一回やれば勝てる」感の醸成） |

### 4-5. モグラたたき — エンドレスモードのスコア記録

**現状の問題:** エンドレスモード（∞）があるのに、スコアが保存されない（未確認だが可能性あり）。

- ゲーム終了時に「今日のベスト: {N}点」「歴代ベスト: {N}点」を並列表示

## 付録：実装時の共通パターン

### 「初回プレイ済み」フラグの書き方（全ゲーム共通）

各ゲームの初期化処理の先頭に1行追加するだけ。

```js
// 各ゲームの script.js 先頭付近
localStorage.setItem('played_minesweeper', '1'); // ゲームIDは各自変更
```

### 難易度事後提案の共通モーダルパターン

```js
function suggestDifficulty(won, currentDiff) {
  const levels = ['easy', 'normal', 'hard', 'oni'];
  const labels = { easy: 'かんたん', normal: 'ふつう', hard: 'むずかしい', oni: '🔴 鬼' };
  const idx = levels.indexOf(currentDiff);
  const suggestion = won ? levels[idx + 1] : levels[idx - 1];
  if (!suggestion) return null;
  return {
    label: labels[suggestion],
    value: suggestion,
    message: won ? `上手い！${labels[suggestion]}に挑戦する？` : `${labels[suggestion]}で試してみる？`
  };
}
```
